📜 Protocolo de Identidad y Diseño
Lead Architect: Camilo Martinez Galarza

Asistente de Arquitectura: Gemini/Cami

"La interfaz es la cara de FoodMaster. Debe ser intuitiva, funcional y profesional."

1. Identidad Visual
Paleta de Colores: Basada en la identidad de marca (Rojos #FF0000, Naranjas, Blancos limpios).

Tipografía: Tipografía Sans-serif moderna, priorizando la legibilidad en pantallas táctiles y entornos de cocina.

2. Componentes UI (Bootstrap 5)
Tablas: Obligatorio el uso de table table-striped table-hover para facilitar la lectura de comandas y pedidos.

Botones: Uso semántico estricto (btn-primary para acciones principales, btn-success para confirmar, btn-danger para cancelar).

Responsividad: El diseño es mobile-first. Todo componente debe ser totalmente operativo en tablets de cocina y dispositivos móviles.

3. UX (Experiencia de Usuario)
Feedback: Todo proceso de espera debe mostrar un spinner o indicador de carga visual.

Gestión de Errores: Mensajes de error amigables pero técnicos (sin exponer estructura del sistema).

Dualidad de Flujo: La UI debe diferenciar claramente entre la gestión de comandas (mesa) y el procesamiento de pedidos (domicilio).

"Diseño funcional para resultados rápidos. 🎨🚀"
